# Estudo de Caso: Aplicando CALMS e as Três Maneiras do DevOps

## Sobre o Projeto

Este projeto apresenta um estudo de caso baseado no desafio proposto pela Rocketseat para aplicação dos conceitos de DevOps em uma empresa fictícia de desenvolvimento de software.

O objetivo é analisar o cenário apresentado, identificar oportunidades de melhoria e propor soluções alinhadas aos princípios do modelo CALMS e das Três Maneiras do DevOps.

## Objetivos

* Identificar problemas culturais e operacionais.
* Propor melhorias baseadas em automação.
* Definir métricas para acompanhamento dos resultados.
* Incentivar o compartilhamento de conhecimento.
* Aplicar os conceitos das Três Maneiras do DevOps.

## Cenário

A empresa fictícia Tech possui equipes de Desenvolvimento e Operações que trabalham de forma isolada, com processos manuais de deploy, testes e monitoramento.

Esse cenário gera atrasos, falhas operacionais e aumento do tempo de recuperação de incidentes.
